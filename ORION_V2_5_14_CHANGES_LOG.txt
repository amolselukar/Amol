1. 7-TIER PROFIT LOCK LADDER (replaces old 3-tier ratchet)
   Location: Line ~731 (config), Line ~2361 (check_exits)
   Old: Velvet Rope (+15 arm, SL=entry+2) + Tier2 (+24 → entry+12) + Tier3 (+36 → entry+24)
   New: PROFIT_LOCK_LADDER = [
           (50, 40), (40, 30), (36, 24), (30, 20), (24, 12), (18, 8), (12, 2),
        ]
   Meaning: If peak reaches entry+50, SL locks at entry+40
            If peak reaches entry+40, SL locks at entry+30
            If peak reaches entry+36, SL locks at entry+24
            If peak reaches entry+30, SL locks at entry+20
            If peak reaches entry+24, SL locks at entry+12
            If peak reaches entry+18, SL locks at entry+8
            If peak reaches entry+12, SL locks at entry+2
   Logic: Sorted highest-first, first match wins. Only ratchets UP (never down).
   Source: Inspired by VRL (Vishal Rajput Trade v20) tick-based tiered exits.

2. VWAP TRIPLE CONFIRMATION ENGINE (futures-first)
   Location: Line ~1908 (check_vwap_signal function)
   Old: Double confirmation — spot VWAP cross + option VWAP
   New: TRIPLE confirmation:
     Gate 1 (PRIMARY): Nifty FUTURES 15m close crosses futures VWAP
       - Body must be >= 50% of candle range
       - Bullish: close > futures VWAP and close > open (green candle)
       - Bearish: close < futures VWAP and close < open (red candle)
     Gate 2: Nifty SPOT 15m close agrees with direction
       - Bullish: spot close > spot VWAP
       - Bearish: spot close < spot VWAP
     Gate 3: ATM option LTP > option's own VWAP (CE for bull, PE for bear)
   Rationale: Futures lead spot by 2-30 minutes (confirmed from manual trading).

3. NIFTY FUTURES TOKEN AUTO-RESOLUTION

   Logic: Fetches NFO instruments, filters name=="NIFTY" + instrument_type=="FUT",
          picks nearest expiry, returns token. No hardcoded token.
   Used by: fetch_nifty_fut_15m() at line ~1264, called in main loop at line ~3196.
